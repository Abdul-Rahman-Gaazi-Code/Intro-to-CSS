
**HTML & CSS Fundamentals Challenge**

#
A challenge to really understand how HTML and CSS work not just copy code, but actually know why things work the way they do.

**What I Worked Through**

- HTML Structure
  ∙ How HTML is the skeleton of every webpage
  ∙ Semantic elements — using the right tag for the right job
  ∙ Forms, links, images, tables, headings, paragraphs

  #

- CSS Styling
  ∙ How to connect CSS to HTML the right way (external, internal, inline)
  ∙ Why external CSS is almost always the best choice

  ∙ The four color formats — Named, Hex, RGB, and HSL — and when to actually use each one.

  ∙ Fonts — Google Fonts, fallbacks, and why they matter
  ∙ Selectors targeting elements with type, class, and pseudo-class selectors like p:first-of-type.
  - The Cascade — why order in your CSS file matters
    ∙ Specificity — why a class beats a plain tag selector
    ∙ Typography — font-size, line-height, letter-spacing, word-spacing, and how they affect readability.

    #

    **What I Actually Learned**

    CSS isn’t just decoration, it’s a set of rules with logic behind it. Once you understand the cascade and specificity, styling stops feeling like guesswork and starts making sense.
